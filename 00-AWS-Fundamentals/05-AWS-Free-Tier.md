# 📖 What is AWS Free Tier?

The **AWS Free Tier** allows new and existing users to explore and learn AWS services at **no cost**, within specific usage limits.

It is designed for:

- 🎓 Students
- 👨‍💻 Beginners
- ☁️ Cloud Engineers
- 🧪 Developers
- 🚀 Startups

The Free Tier lets you build and test applications without purchasing expensive infrastructure.

> **Important:** The Free Tier is not unlimited. Charges may apply if you exceed the allowed limits or use services that are not covered.

# ⭐ Why Use AWS Free Tier?

The AWS Free Tier helps you:

- Learn AWS without large costs
- Practice real-world cloud scenarios
- Prepare for AWS certifications
- Build personal projects
- Gain hands-on experience

It is the best way to start your AWS journey.

# 📦 Types of AWS Free Tier Offers

AWS provides three types of Free Tier offers.

## 1️⃣ 12-Month Free Tier

Available for **12 months** after creating a new AWS account.

Examples include:

- Amazon EC2
- Amazon S3
- Amazon RDS

---

## 2️⃣ Always Free

These services have free monthly usage limits that continue even after the first 12 months.

Examples include:

- AWS Lambda
- Amazon DynamoDB
- Amazon CloudWatch
- AWS IAM

---

## 3️⃣ Free Trials

Some AWS services offer a limited free trial period.

Examples:

- Amazon SageMaker
- Amazon Redshift
- Amazon QuickSight

The trial duration varies by service.

# 🚀 Popular Free Tier Services

| Service | Free Tier Benefit |
|----------|-------------------|
| Amazon EC2 | Limited free compute usage for eligible accounts |
| Amazon S3 | Limited free object storage |
| Amazon RDS | Limited free database usage |
| AWS Lambda | Free monthly requests and compute time |
| Amazon DynamoDB | Free monthly storage and read/write capacity |
| AWS IAM | Always Free |
| Amazon CloudWatch | Free monitoring limits |

> **Note:** Free Tier limits can change over time. Always check the AWS documentation for the latest details.

# ⚠️ Tips to Avoid Unexpected Charges

Follow these best practices:

- Stop EC2 instances when not in use.
- Delete unused EBS volumes and snapshots.
- Remove unused Elastic IP addresses.
- Delete unused load balancers.
- Monitor your usage regularly.
- Set up AWS Budgets and billing alerts.
- Read the Free Tier limits before using a service.

These practices help keep your AWS account under control.

# 🌟 Real-World Example

Imagine you're learning AWS.

You create:

- One EC2 instance for practice

As long as your usage stays within the Free Tier limits (where applicable), you can practice these services without paying.

If you launch multiple large resources or exceed the Free Tier limits, AWS may charge for the additional usage.
